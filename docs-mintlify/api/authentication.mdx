# Authentication

All API endpoints require a valid JWT bearer token.

## Getting Started

### 1. Login

Send your credentials to get a JWT token:

```bash
curl -X POST http://localhost:3000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "password": "password"
  }'
```

**Request Body:**
```json
{
  "username": "string",
  "password": "string"
}
```

**Response (200 OK):**
```json
{
  "message": "Login successful",
  "role": "admin",
  "auth_session": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
}
```

**Error Response (401):**
```json
{
  "error": "Invalid credentials"
}
```

### 2. Use Token

Include token in Authorization header:

```bash
curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  http://localhost:3000/api/category-data?category=fruits&pincode=400706
```

## Token Details

- **Type:** JWT (JSON Web Token)
- **Algorithm:** HS256
- **Expiration:** 7 days
- **Storage:** HttpOnly cookie (set automatically)
- **Format:** `Bearer <token>`

## Available Credentials

**Admin User:**
- Username: From `ADMIN_USERNAME` env var
- Password: From `ADMIN_PASSWORD` env var
- Role: Full access to all endpoints

**Regular User:**
- Username: From `USER_USERNAME` env var  
- Password: From `USER_PASSWORD` env var
- Role: Limited access (read-only)

## Logout

Clear your session:

```bash
curl -X POST http://localhost:3000/api/auth/logout \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

**Response (200):**
```json
{
  "message": "Logout successful"
}
```

## Security Tips

✓ **DO:**
- Store tokens securely
- Use HTTPS in production
- Set tokens as httpOnly cookies
- Expire tokens after 7 days
- Rotate credentials regularly

✗ **DON'T:**
- Hardcode credentials
- Log tokens
- Share tokens
- Use expired tokens
- Commit credentials to version control

## Environment Variables

Configure credentials via environment variables:

```bash
# .env
ADMIN_USERNAME=admin
ADMIN_PASSWORD=secure_password_123
USER_USERNAME=reader
USER_PASSWORD=reader_password_123
```

## Token Validation

Tokens are validated on every request. Invalid tokens return 401:

```json
{
  "error": "Invalid or expired token",
  "status": 401
}
```

## Troubleshooting

**"Invalid credentials"**
- Check username/password
- Verify environment variables are set
- Ensure you're using correct endpoint

**"Invalid or expired token"**
- Token has expired (get new one)
- Token is malformed
- Secret key mismatch

**"Unauthorized"**
- No Authorization header provided
- Wrong Authorization format
- User role lacks permission
